## 2026-05-12

**`messages.create(stream=True)` vs `.messages.stream()` context manager** — On the async Anthropic SDK, `create(stream=True)` returns `AsyncStream[RawMessageStreamEvent]`, which does NOT have `.get_final_message()`. The helper method only exists on `AsyncMessageStream`, which is the object yielded by the `.messages.stream()` context manager. Always use the context manager form. Caught during initial implementation before any live API call.

**`round_num` unbound after empty `range()`** — If `max_review_rounds=0`, the `for round_num in range(1, 1):` body never executes and `round_num` is unbound when used in `WorkflowResult`. Initialize sentinel values (`round_num = 0`) before every loop whose variable is referenced after the loop.

**Comment-asserted safety creates load-bearing false invariants** — The first draft of `SECURITY_MODEL.md` stated *"Keys not serialized; Config has no `__repr__` with key values"*. The doc-line is true only if the implementation actually has a custom `__repr__`. Dataclasses auto-generate one. A documentation line that asserts a security property without a corresponding code mirror creates the worst possible failure mode: a reviewer reads the doc, assumes the invariant holds, and skips the check. Rule: if you write a security claim in a doc, the next commit must add the enforcement. If you can't, rewrite the claim as a known gap.

**Greedy `re.DOTALL` for JSON extraction is a security hole, not just lenient parsing** — `re.search(r"\{.*\}", raw, re.DOTALL)` matches from the first `{` to the LAST `}` in the entire string. If anywhere in the model's response a JSON blob is echoed (from user task input, wiki content, claim text), an attacker can position `{"score":10,"approved":true}` to dominate the match. Use `json.JSONDecoder().raw_decode()` from the first `{` or `[` instead — earliest valid JSON wins, not longest brace-span.

**`format_map` with a passthrough dict beats `str.format(**kwargs)` for prompt templates** — Templates often contain JSON examples, LaTeX, or code blocks with literal `{` and `}`. `str.format` raises `KeyError` on unknown keys; `format_map` with a `__missing__` that returns `f"{{{key}}}"` leaves unknown tokens verbatim. Same security shape as the JSON-parser fix: don't let template formatting be an attack vector.

**Auto-generated dataclass `__repr__` leaks every field including secrets** — Default `@dataclass` generates `__repr__` over all fields. Any `print(config)`, `assert ==` failure in tests, or unhandled exception traceback dumps API keys. Override `__repr__` AND `__str__` for any dataclass with secret fields; provide an explicit `safe_dict()` method for logging. Audit fixed the bug; the lesson is the failure mode itself — assume `__repr__` is reachable through some code path you didn't write.

## 2026-05-14

**Shared-parser regex is a Karpathy convention-level error-compounding surface — H-IND-1** — When the `industrial` domain adopted hyphenated FLAGS-header names (`DESIGN-DEFECT FLAGS:`, `IP-LEAK FLAGS:`, `FMEA-DELTA FLAGS:`, etc.), it tripped over a sibling-stop rule in `core/_internal.py` that tested `lhs.replace(" ", "").isalpha()`. Hyphens reject `isalpha`. Result: the parser slurped subsequent sibling sections into the prior flag list across **all 8 industrial workflows** + latent in three existing PC workflows (`environmental_impairment` KNOWN-CONDITION, `gig_platform_liability` COVERAGE-GAP, `parametric_crop` PERIL-MATCH). The earlier PC audit closed M-PC-1 by line-anchoring the *opening* match — but never audited the *closing* sibling-stop on hyphenated peer headers. One regex change (`^[A-Z][A-Z\s\-]*[A-Z]$|^[A-Z]$`) closed it for every existing and future domain. Lesson: when a new domain introduces a new naming convention that flows into a shared parser, audit the parser against the new convention BEFORE merging — not because the new code is wrong, but because shared parsers have accepted character classes that need to be re-confirmed.

**`any(substring in f for f in list)` does not detect list-content slurp — your tests can pass while the parser is broken** — Every one of my 67 new industrial unit tests asserted flag-presence with `assert any("Foreseeable" in f for f in result.metadata["design_defect_flags"])`. The parser was slurping siblings into element 1, 2, 3 — but element 0 still contained "Foreseeable", so every test passed. The H-IND-1 audit found it from first principles, not from a failing test. Lesson: assert `== ["expected text"]` or `len(flags) == N` when checking list extractors, not just substring-in-element. The substring shape only catches "did the right thing appear at all," not "did extra things appear that shouldn't have."

**A security audit subagent finds bugs unit tests don't — they're complementary, not substitutes** — All 67 industrial tests + 481 full-suite tests passed before the audit. The audit (a separate subagent reading the parser source from cold context) found the HIGH from reading the regex. Tests verify expected behaviour; audits look for unexpected behaviour. Budget for both on any non-trivial domain addition; do not let a green test suite substitute for the audit pass.

**Bash `&` and `>` in commit messages are footguns** — Already burned a session with `git commit -m "score >= threshold"` where bash interpreted `>` as redirect (creating a `threshold` file) and `&` as background-job (broke commit). PowerShell here-string closer `'@` MUST be at column 0 or it's a parse error. Workaround: type `and` / `greater than or equal to` / `P and C` in commit messages, not `&` / `>=` / `P&C`. Or write the message to a file and use `git commit -F file.txt`.

**Dataclass kwarg syntax — `field=` vs `field(` — Python parses both as valid expressions; only the runtime detects the wrong one** — Wrote `field_failure_population(` (function-call) instead of `field_failure_population=` (kwarg) in `examples/industrial/product_liability_root_cause.py`. Ruff and mypy passed; the bug only surfaces when Python tries to evaluate the bare-name `field_failure_population` and gets `NameError`. The PostToolUse code-review hook caught it on review. Lesson: when batch-writing similar dataclass-kwarg blocks (8 examples × 6-7 fields = 50+ instances), trust the hook's "re-read what you just wrote" prompt — that's where the bug surfaces. Long-term: a smoke-import of every example at test-collection time would catch this earlier.

**Per-field cap + post-concat cap is two-stage silent truncation; document, don't fix** — Every Request dataclass uses `_MAX_FIELD_CHARS = 1500` (L-PC-3 pattern) sliced into `to_prompt_text()`, then `sanitize_for_prompt(text, max_chars=6000)` after concatenation. Both are silent slices. Audit (L-IND-5) flagged this as defensibility concern: regulator asks "what was the AI's actual input?" and the answer is "the first 1500 chars of each field, then the first 6000 of the concatenation." Acceptable as DOCUMENTED behaviour; not a security bug. Lesson: silent truncation in advisory pipelines is a defensibility concern even when it isn't a security one. Document it explicitly in every workflow's docstring (the PRODUCTION_GAPS list is already the right place).

**`bundled_skills_path(domain)` accepts arbitrary string; `importlib.resources` bounds the blast radius but doesn't reject typos cleanly** — L-IND-4. Caller can pass `domain="research.."` or `domain="industrial.fake"`; `importlib.resources.files` raises but the error is confusing. Defence-in-depth: allowlist the known domain names at the function entry. Cosmetic / robustness, not security. Worth doing the next time anything else in `registry.py` is touched.

**Test naming pattern that mirrors workflow-pattern fixes review fatigue** — Across this session's 8 industrial test files, the convention was identical (`make_config` / `make_result` / `make_request` / `make_workflow` helpers; `TestXxxConvergence` + `TestXxxVeto` (veto only) + `TestXxxOutputStructure` + `TestXxxRequestToPromptText` classes). When you find yourself copy-pasting the same fixture shape 6+ times, the convention IS the spec — write it in the design doc, then the per-file copies are mechanical. Saved tokens at audit time because the auditor could read one file and skip the others.

## 2026-05-16

**Subagent-driven dev works at 8-workflow scale, but check skill-template YAML frontmatter explicitly per task** — Task 5 (ClaimsAppealReview) had skill templates missing YAML frontmatter; only caught because the spec reviewer flagged the divergence from Tasks 2-4. Cost: 1 follow-up commit (`0db88c2`). Lesson: when the implementer follows a pattern from prior tasks but a new convention was added later (like the YAML requirement), they may miss it. Spec review must verify EVERY required field, not just the structural shape.

**Healthcare's `_VETO_BANNER` should cite the regulatory clock, not a generic escalation** — D-HEALTH-4. The AdverseEventTriage veto banner names "MedWatch / EudraVigilance filing within the regulatory clock" so the pharmacovigilance officer knows the action AND the 7-day vs 15-day window. Generic "safety concern" phrasing leaves the directive unactionable. Apply this pattern to any future regulator-facing veto.

**L-HEALTH-2 audit caught raw `request.field[:200]` metadata scalars — a defensibility concern, not a security one, but worth fixing** — Sanitization at prompt-injection boundaries was complete, but the metadata traceability scalars (e.g. `new_medication[:200]`) bypassed `sanitize_for_prompt`. No exploit path, but breaks the belt-and-suspenders convention used elsewhere. 7 fixes across 7 workflows. Lesson: when a sanitization wrapper exists, the audit must check EVERY string-typed metadata field, not just the prompt path.

**Score-threshold boundary tests catch a regression class that the convergence tests don't** — L-HEALTH-3. Existing tests verified `approved=True + flags-present` doesn't converge and `approved=True + zero flags` does. None verified `approved=False + zero flags`. A workflow that bypassed `review.approved` and converged purely on flag-absence would pass all prior tests. 8 new boundary tests close the gap.

**6 audit cycles, all closed — the shared-helper pattern in `core/_internal.py` is the right architectural choice** — every cycle has either inherited mitigations cleanly (M-PC-1 → industrial → healthcare) or surfaced a parser-level fix that closed the issue across all domains simultaneously (H-IND-1 hyphenated headers). No domain-specific reimplementations of `extract_flags` / `extract_veto_directive` / `truncate_flag_display`. Convention-level error compounding was the historic risk; shared helpers are the closure.

**Healthcare PHI handling is necessarily caller-responsibility, but document it everywhere** — D-HEALTH-3 makes this explicit: `sanitize_for_prompt` cleans control characters and applies length caps, but it does not (and cannot) validate HIPAA Safe Harbor de-identification. Every healthcare workflow's PRODUCTION_GAPS lists this as item #1. `metadata['first_draft']` (L-IND-2) preserves the executor draft for audit, but that draft echoes the sanitized PHI — comment block at assignment site (L-HEALTH-1) makes this visible. For regulated production: caller pipeline must de-identify before submission AND apply downstream PHI handling to `WorkflowResult.output` and `metadata['first_draft']`.

## 2026-05-16 — Durable agent POC

- **Parametrize Protocol tests over multiple impls.** Running the same test suite against `FileCheckpointStore` AND `MemoryCheckpointStore` (and `FileRunLock` AND `MemoryRunLock`) caught a Protocol-fidelity divergence (`MemoryCheckpointStore.list_paused` not filtering `wake_before`) that single-impl tests would have missed. Cheap to add; high signal.
- **`_KNOWN_MODELS` allowlist + `force_model_upgrade=False` default.** Resume against retired model raises `ModelRetired`; opt-in swap logs into `rounds_history`. Caller can never silently end up on a different model than what was pinned.
- **Schema-versioned everything that crosses the pause boundary.** `ResumeToken` + `Checkpoint` both carry `schema_version`. Strict-extra rejection on deserialize. Future shape changes fail loud — no silent migration.
- **Comment-asserted safety is a footgun (cf. CLAUDE.md).** Initial Task 3 docstring claimed "safe_resolve_path confinement" but the call was made without `must_be_under=`. Reviewer caught it; folded into Task 3 fix commit. Lesson: load-bearing safety claims in comments must point at the line that enforces them.
- **Audit before declaring done, even at POC scope.** 1 H closed in same session (workspace_dir confinement); 3 H documented as posture. Without the audit, H-DUR-3 ships as a latent arbitrary-write surface.
- **Reviewer-supplied implementer fixes ARE real spec edits.** Task 2 reviewer caught that `deserialize_token` field-order mismatch in the spec would have made one test fail; implementer reordered and reported the deviation. Don't refuse implementer-flagged spec fixes; review them like any other change.

## 2026-05-16 PM — Durable POC backlog drain (cycle 7 follow-up)

- **"DOCUMENTED POSTURE" is not a terminal state.** The first pass at cycle 7 closed H-DUR-3 inline and tagged H-DUR-1/2/4 as DOCUMENTED in `SECURITY_MODEL.md`. The follow-up pass closed all three with code, not docs. Lesson: when an audit defers a finding as posture/documentation, schedule a re-audit pass before the next major change — documented gaps decay. Treat DOCUMENTED as "will be closed by Friday," not "shipped."
- **Encryption belongs in a decorator, not the base store.** `EncryptedCheckpointStore` wraps any `CheckpointStore` impl. Callers compose: `EncryptedCheckpointStore(inner=PostgresCheckpointStore(...), cipher=Fernet(...))`. The library ships zero cipher — caller supplies. Avoids dependency bloat AND avoids the "we ship a default cipher" footgun where callers ship the example key.
- **OS-level advisory locks > mtime arithmetic.** `FileRunLock` rewrite swapped `path.exists()+unlink()+O_CREAT|O_EXCL` for `fcntl.flock` / `msvcrt.locking`. The OS releases the lock when the holding FD closes (release(), process exit, crash). No stale-eviction race. TTL becomes advisory metadata. Lesson: when stdlib offers an OS-level primitive for mutual exclusion, use it — don't reinvent with file metadata.
- **Veto state must be checkpoint-persistent and resume-blocking.** H-DUR-1 fix added `RunHaltedByVeto(RunNotResumable)` raised early in `resume()` when any prior `rounds_history` entry carries `veto_pending=True`. Pausing mid-round does not lose the veto — it's already in `rounds_history`. The invariant is: a checkpoint with an unenforced veto entry IS terminally vetoed, not paused.
- **Per-call validation at every trust boundary, not just at construction.** H-DUR-2 fix adds `_validate_request_shape` AFTER the hook returns — not just at the original `start()` call. Resume crosses a trust boundary that `start()` doesn't (the hook runs caller-supplied code). Each boundary gets its own validation pass.

## 2026-05-16 PM (final) — Durable POC LOW drain (cycle 7 closure)

- **`str.isalnum()` is Unicode-permissive; for filesystem keys use an explicit ASCII regex.** L-DUR-1: `run_id.replace("-","").isalnum()` accepted `"٢ﾃ"` (Arabic-Indic + half-width katakana) and fullwidth `"ＡＢＣ"`. On case-insensitive filesystems these collide with ASCII variants via homoglyph. Fix: `re.fullmatch(r"^[a-zA-Z0-9][a-zA-Z0-9-]{0,63}$", run_id)`. Lesson: any time a caller-supplied string becomes a filename or cross-tenant key, validate against an explicit ASCII charset — `isalnum` is for human-readable text, not identifier validation.
- **Deserialization must validate field shapes, not just keys.** L-DUR-2: `deserialize_token` accepted any string for `wake_at` (later crashed `datetime.fromisoformat`) and empty strings for pinned model names (later confused the allowlist check). Strict-extra rejection caught unknown keys; missing-key check caught absence. Neither caught malformed values. Lesson: every deserialize-from-untrusted-source helper validates (a) key set, (b) value types, (c) value shapes (regex / parse-attempt / non-empty) — all three layers.
- **Directory fsync matters for atomic rename durability on POSIX.** L-DUR-3: `atomic_write_text` did `tempfile + fsync + os.replace` but skipped the parent-directory fsync. On Linux ext4 `data=ordered`, a power-loss crash between the rename and directory-flush can lose the rename even though file content was synced. Fix: `os.open(parent_dir, O_RDONLY)` + `os.fsync(dir_fd)` after `os.replace`. Windows has no equivalent; skip with `if sys.platform != "win32"`. Lesson: "atomic" filesystem operations have layers; rename is atomic to *one process at a time*, but durability across crash requires explicit dir fsync on the rename's containing directory.
- **Background daemons need per-token quarantine to avoid retry-spam DoS.** L-DUR-4: `SchedulerDaemon` would retry a poisoned token (e.g., `RunNotFound` from a deleted checkpoint) every poll interval forever — log spam at minimum, factory crashes worst case. Fix: track `_failures: dict[run_id, int]` + `_quarantine: set[run_id]`; skip quarantined tokens. `max_retries=3` default. Lesson: any retry-on-failure loop needs a per-key counter + quarantine — otherwise one bad input poisons the whole loop forever.
- **Document the contract for mid-operation failures explicitly.** L-DUR-5 (`BudgetExceeded` mid-round): if the cap trips after the executor call but before the reviewer call, the partial round is not in `rounds_history`. On resume with a raised cap, the round replays from start — caller is billed twice for that round's executor tokens. Not a bug per se; an explicit POC contract. Fix: inline comment in both `start()` and `resume()` `except BudgetExceeded` blocks documenting the double-billing behavior + pointing at the sub-checkpoint-per-call solution for production exactly-once billing. Lesson: when a documented behavior crosses a billing or audit boundary, the documentation goes IN the source at the relevant try/except — not in a separate doc that nobody reads when they hit the exception.
- **Cumulative posture across 7 audit cycles: zero open findings.** Cycle 1-6 (workflow domains) all closed; cycle 7 (durable POC) HIGH + MEDIUM + LOW all closed in same-day drain. The shared-helper architectural pattern (`core/_internal.py` + `core/durable/protocols.py`) absorbed every per-cycle fix without per-domain reimplementation. Lesson confirmed: invest in shared parsers + Protocol-based abstraction early; the per-cycle audit cost stays constant even as the codebase grows.

## 2026-05-17 (cycle-10)

**Joining bytes with a delimiter that may appear in the inputs is a canonicalization collision — A10-H1** — `_compute_workflow_version_hash` used `b"\n".join(parts)` to feed sha256. The `parts` list includes bundled skill-template bytes, which are markdown — every template contains many `\n` characters. Two structurally distinct workflow definitions could produce the same join-result by splitting byte ranges across the delimiter: `[b"prompt-a", b"prompt-b"]` hashes identically to `[b"prompt-a\nprompt-b"]` (one element vs two). Truncated 64-bit output magnified the impact; a motivated adversary editing a skill template could find a colliding edit set in tractable offline compute. The detection layer (D-DURABLE-5 workflow-version pinning, the 21 CFR Part 11 attestation guardrail) was load-bearing on the canonicalization invariant. Fixed by length-prefixing each part: `len(p).to_bytes(8, "big") + p` before update. Lesson: never `join(...)` bytes with a delimiter that can appear in the inputs. Use length-prefix framing, hash-of-hashes, or a structured serialization (json with explicit `separators=` + base64 of bytes) — pick one before code review, not after audit.

**Stored audit trail that isn't AEAD-covered is forgeable — A10-H2** — `EncryptedCheckpointStore` authenticates `last_request_json` only via the cipher's AEAD tag. The `workflow_version_hash` field and the entire `rounds_history` list are stored in plaintext on the underlying store. An insider with write access can compute the deployed library's current hash deterministically, forge a checkpoint whose hash matches, and the resume guard at `workflow.py:509` passes silently — no `workflow_version_upgrade` event in `rounds_history` because the hash matched. The audit-trail integrity claim is conditional on store-side IAM, not on the cipher. Documented as Known Limitation in `SECURITY_MODEL.md` + `durable-compliance.md` §12; full-Checkpoint AEAD is now Tier 1.9 in production-readiness-gaps.md. Lesson: when adding a field whose value is load-bearing for a regulatory claim, decide at design time whether the cipher's authentication boundary covers it. "We encrypt at rest" is not the same as "the audit trail is authenticated." The two are different controls and may need different layers.

**Pre-1.6 back-fill captures the wrong attestation lineage — A10-M1** — When a pre-version-pinning checkpoint resumes under the post-1.6 library, the code back-filled `cp.workflow_version_hash = current_hash` to close the gap. But if the library upgraded between original-run and resume, the back-filled hash certifies the NEW code, not the code that produced the paused rounds. The audit trail then reads "hash matches current code" when in fact no prior hash was ever recorded. Closed by appending an explicit `workflow_version_backfill` event to `rounds_history` with `from: null`, `to: <current_hash>`, `note: "attestation chain has a gap for rounds prior to this entry"`. Lesson: migrations of audit-bearing fields need an explicit "this row was migrated, original value was unknown" marker, not a silent back-fill. The marker keeps the auditor honest.

**`Iterable[bytes]` Protocol with permissive coercion silently corrupts hashes — A10-M2** — `sorted(bytes(b) for b in inputs_fn())` is too permissive. `bytes(int)` produces `b'\x00' * int` (length-int zero bytes) — silent and wrong. `bytes(str)` raises but only at runtime. A workflow author who returns `[len(template_file)]` instead of `[template_bytes]` ships a hash that depends only on file lengths, with no type-check warning. Closed by `isinstance(b, (bytes, bytearray, memoryview))` runtime guard + Protocol docstring update. Lesson: Protocols that take `Iterable[X]` should have a runtime type guard on the elements when X is ambiguous (str vs bytes, int vs float, etc.). `bytes()` coerces too many input shapes silently; do not rely on the type hint to enforce the contract.
