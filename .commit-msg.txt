fix(security): MCP DoS hardening H3 H4

Bounds caller-supplied inputs to Skill.render at the class-method
chokepoint so every consumer (MCP server, programmatic library
caller, tests) inherits the same guarantees. Single-path-of-control
per CLAUDE.md.

H3 (unbounded skill input values via MCP): per-value length capped
at 8192 chars; non-string values coerced via str().

H4 (MCP render_skill DoS + control-char passthrough): key count
capped at 64; control chars (incl. ANSI ESC 0x1b, BOM 0x7f, CSI
0x9b) stripped from every value before format_map substitution.
Prevents terminal-injection on downstream consumers and bounds
worst-case prompt size.

Tests: +3 unit tests covering oversized value, key flood, and
control-char strip. 215 -> 218 passing.

Stacks on fix/security-hardening-pre-sprint (PR #5).
